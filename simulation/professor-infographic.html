Secure Microservice Deployer

Deploy, protect, and validate microservices on Kubernetes through one controlled gateway.

Architecture Story

Frontend
(NodePort 30000)
→
Deployer Backend
(ClusterIP)
→
K8s User Service
(ClusterIP)
→
Gateway
(NodePort 30080)

Key policy: user services stay internal; only gateway is public.

Security Controls

WAF: SQLi/XSS Header Sanitization Rate Limiting API-Key Routes WAF Event Logging

Operational Pipeline

  • Upload Docker image tar
  • Deploy service into `user-services` namespace
  • Auto-register route in gateway
  • Enable WAF / Rate Limit / API key per service
  • Run Trivy (image) + Nikto (runtime) scans
  • Review logs and security stats from Redis-backed endpoints

Demonstrable Proof Points

  • Malicious payload blocked with HTTP 403 (WAF)
  • Flood traffic blocked with HTTP 429 (Rate Limiter)
  • Protected route returns HTTP 401 without valid key
  • Vulnerability findings visible via Trivy and Nikto results
  • WAF events and counters persisted for auditability
